# 404エラー修正完了

## 問題

ログイン時に以下のエラーが発生していました：
```
POST http://localhost:3001/auth/admin/login 404 (Not Found)
```

## 原因

フロントエンドのAPI設定で、`baseURL`が`http://localhost:3001`になっていたため、`/api`プレフィックスが抜けていました。

バックエンドのルーティングは`/api/auth/admin/login`ですが、フロントエンドは`/auth/admin/login`にリクエストを送っていました。

## 修正内容

`frontend/lib/api.ts`の`baseURL`を以下のように修正しました：

**修正前**:
```typescript
baseURL: API_URL,  // http://localhost:3001
```

**修正後**:
```typescript
baseURL: `${API_URL}/api`,  // http://localhost:3001/api
```

## 確認方法

1. **ブラウザのページを再読み込み**（F5）
2. **ログインページで再度ログインを試す**
   - 店舗ID: `sample-store-001`
   - パスワード: `password123`

3. **開発者ツール（F12）のネットワークタブで確認**
   - `/api/auth/admin/login` にリクエストが送られているか確認
   - ステータスコードが200になっているか確認

## 修正後の動作

- 正しいURL: `http://localhost:3001/api/auth/admin/login`
- ログイン成功後、ダッシュボードにリダイレクトされます

## 注意事項

開発サーバーを再起動する必要はありませんが、ブラウザのキャッシュをクリアすることを推奨します：
- `Ctrl + Shift + Delete` → キャッシュをクリア → ページを再読み込み
