# pgAdmin警告の対処方法

PostgreSQLのインストール中に「pgAdminサーバーをシャットダウンしてください」という警告が表示された場合の対処方法です。

## 対処手順

### ステップ1: pgAdminを終了

1. **タスクバーを確認**
   - タスクバーの通知領域（時計の近く）にpgAdminのアイコンがあるか確認
   - アイコンを右クリックして「終了」を選択

2. **タスクマネージャーで確認**
   - `Ctrl + Shift + Esc` でタスクマネージャーを開く
   - 「プロセス」タブで「pgAdmin」を検索
   - 見つかったら右クリック→「タスクの終了」

3. **pgAdminアプリケーションを閉じる**
   - pgAdminが開いている場合は、すべてのウィンドウを閉じる

### ステップ2: PostgreSQLサービスを確認

```powershell
# PowerShellを管理者として実行して、サービスを確認
Get-Service -Name postgresql*
```

### ステップ3: 警告ダイアログで「OK」をクリック

pgAdminを終了したら、警告ダイアログで「OK」をクリックしてインストールを続行してください。

## 代替方法：pgAdminを無効化してインストール

pgAdminが不要な場合（コマンドラインでPostgreSQLを使用する場合）：

1. PostgreSQLインストーラーの「コンポーネント選択」画面で
2. 「pgAdmin 4」のチェックを外す
3. インストールを続行

## インストール後の確認

PostgreSQLのインストールが完了したら：

```powershell
# PostgreSQLに接続できるか確認
psql -U postgres
```

パスワードを求められたら、インストール時に設定したパスワードを入力してください。

## トラブルシューティング

### pgAdminが終了しない場合

1. **強制終了**
   - タスクマネージャーで「pgAdmin」プロセスを強制終了

2. **再起動**
   - コンピューターを再起動してから、再度インストールを実行

### PostgreSQLサービスが起動しない場合

```powershell
# サービスを開始（管理者権限で実行）
Start-Service postgresql-x64-16
```

サービスの名前は、インストールしたPostgreSQLのバージョンによって異なります（例: `postgresql-x64-16`）。
