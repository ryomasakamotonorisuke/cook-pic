# 開発環境ログイン情報

## 📋 概要

このドキュメントには、Pic_cul開発環境で使用するすべてのログイン情報が記載されています。

---

## 🔐 管理者ログイン情報

### 管理者ログインページ
- **URL**: http://localhost:3000/admin/login
- **店舗ID**: `sample-store-001`
- **パスワード**: `password123`

### アクセスできる機能
- 日間メニューの投稿・編集・削除
- 週間メニューの設定
- 月間メニューの設定
- 店舗プロフィールの編集
- メニュー一覧の確認

---

## 👤 ユーザーアクセス情報

### ユーザーアクセスポイント
- **URL**: http://localhost:3000/user/access
- **店舗ID**: `sample-store-001` を入力

### アクセスできる機能
- 日間メニューの閲覧（フィード形式）
- 週間メニューの閲覧
- 月間メニューの閲覧
- 店舗プロフィールの閲覧

## 🗄️ データベース情報

### PostgreSQL接続情報
- **データベース名**: `pic_cul`
- **ホスト**: `localhost`
- **ポート**: `5432`
- **ユーザー名**: `postgres`
- **パスワード**: `postgres`

### 接続文字列
```
postgresql://postgres:postgres@localhost:5432/pic_cul
```

### pgAdminでの接続
1. pgAdmin 4を起動
2. サーバー「PostgreSQL 18」に接続
3. パスワード: `postgres`
4. データベース「pic_cul」を選択

## 🔧 環境変数

### フロントエンド（`frontend/.env.local`）
```env
NEXT_PUBLIC_API_URL=http://localhost:3001
```

### バックエンド（`backend/.env`）
```env
DB_TYPE=postgres
DATABASE_URL=postgresql://postgres:postgres@localhost:5432/pic_cul
STORAGE_TYPE=local
PORT=3001
JWT_SECRET=dev-secret-key-change-this-in-production
NODE_ENV=development
```

## 🎯 クイックスタート

1. **開発サーバー起動**
   ```powershell
   npm run dev
   ```

2. **ブラウザでアクセス**
   - 管理者ログイン: http://localhost:3000/admin/login
   - 店舗ID: `sample-store-001`
   - パスワード: `password123`

3. **メニューを投稿**
   - ダッシュボードから「日間メニューを投稿」をクリック
   - 画像、メニュー名、価格を入力

4. **メニューを確認**
   - http://localhost:3000/user/access にアクセス
   - 店舗ID: `sample-store-001` を入力
   - 投稿したメニューが表示されます

## ⚠️ 注意事項

- このログイン情報は**開発環境専用**です
- 本番環境では必ずパスワードを変更してください
- PostgreSQLのパスワードは本番環境で強力なパスワードに変更してください
- JWT_SECRETは本番環境でランダムな文字列に変更してください

## 📞 トラブルシューティング

### ログインできない場合
- PostgreSQLサービスが起動しているか確認
- `backend/.env`の`DATABASE_URL`が正しいか確認

### データベース接続エラーの場合
- PostgreSQLサービスを起動: `Start-Service -Name "postgresql-x64-18"`
- パスワードが正しいか確認

### 開発サーバーが起動しない場合
- ポート3000または3001が使用されていないか確認
- `npm install`を実行して依存関係をインストール
